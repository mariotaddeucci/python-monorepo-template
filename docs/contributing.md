# Contributing

## Development Setup

```bash
# Clone and install
git clone https://github.com/mariotaddeucci/python-monorepo-template.git
cd python-monorepo-template
uv sync
```

## Code Style

- Line length: 120 chars
- Indentation: 4 spaces
- Quotes: double (`"`)
- Full type annotations on all public functions and methods
- Use built-in generics (`list[str]`, `dict[str, int]`)
- Google-style docstrings on all public API
- `-> None` required on `__init__`

### Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Modules/functions | `snake_case` | `process_data` |
| Classes | `PascalCase` | `DataModel` |
| Constants | `UPPER_SNAKE` | `MAX_RETRIES` |
| Private | `_leading_underscore` | `_internal_helper` |

## Running Tests

```bash
# All packages
uv run task test

# Single package
uv run --directory packages/my-sample-package-provider-core task test
```

### Test Guidelines

- Functions only (`def test_*`), never classes
- Use `pytest.raises` for expected exceptions
- Test names describe behaviour: `test_clamp_above_max`

## Linting and Formatting

```bash
# Lint
uv run task lint

# Autofix
uv run task autofix

# Format
uv run task format
```

## Adding a New Package

1. Create the package directory under `packages/`
2. Add `pyproject.toml` with hatch-vcs configuration
3. Add `src/<module>/`, `tests/`, `README.md`, `.python-version`
4. Add `mkdocs.yml` and `docs/` for documentation
5. Update root `pyproject.toml` taskipy tasks

## Common Pitfalls

- Never edit `src/<module>/_version.py` — auto-generated by `hatch-vcs`
- Dev deps go in `[dependency-groups] dev` in root `pyproject.toml`, not in `[project.optional-dependencies]`
- Always use `uv` — never `pip`, `python -m pip`, or `poetry`
- Never use `uv run --package X task Y` (causes infinite recursion); use `uv run --directory packages/X task Y`

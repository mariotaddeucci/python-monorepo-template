# AGENTS.md

Guidelines for agentic coding agents working in this Python monorepo.

## Repository structure

```
python-monorepo-template/
├── pyproject.toml          # workspace root: ruff, pytest, taskipy
├── uv.lock
└── packages/
    ├── my-sample-package/                     # proxy package — re-exports all providers
    ├── my-sample-package-provider-core/       # Python >=3.11, module: core
    ├── my-sample-package-provider-app/        # Python >=3.12, module: app
    └── my-sample-package-provider-standalone/ # Python >=3.13, module: standalone
```

Each package: `src/<module>/`, `tests/`, `pyproject.toml`, `.python-version`, `README.md`.

## Toolchain

Always use `uv` — never `pip`, `python -m pip`, or `poetry`.
Never use `uv run --package X task Y` (causes infinite recursion); use `uv run --directory packages/X task Y`.

For all build/test/lint commands, load the `project-commands` skill.

## Code style

- Line length: 120 chars. Indentation: 4 spaces. Quotes: double (`"`).
- Full type annotations on all public functions and methods. Use built-in generics (`list[str]`, `dict[str, int]`).
- Naming: `snake_case` modules/functions, `PascalCase` classes, `UPPER_SNAKE` constants, `_leading_underscore` private.
- Google-style docstrings on all public API. `-> None` required on `__init__`.
- Raise specific exceptions (`ValueError`, `RuntimeError`, etc.) with descriptive messages.
- Never bare `except:`. Never `assert False` — use `pytest.raises` in tests.
- Optional deps guarded with `try/except ImportError`.

## Tests

- Functions only (`def test_*`), never classes.
- Use `pytest.raises` for expected exceptions.
- Test names describe behaviour: `test_clamp_above_max`.

## Versioning

- Dynamic via `hatch-vcs`. Never hard-code versions.
- Tag format: `<package-name>:<version>` — e.g. `my-sample-package-provider-core:1.2.3`.

## Common pitfalls

- Never edit `src/<module>/_version.py` — auto-generated by `hatch-vcs`.
- Dev deps go in `[dependency-groups] dev = [...]` in root `pyproject.toml`, not in `[project.optional-dependencies]`.
